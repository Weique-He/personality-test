一个专业的在线性格测试系统，基于 MBTI 理论，提供详细的性格分析报告。支持密钥验证、云端管理、自动发货等功能。

**在线演示：** https://Weique-He.github.io/personality-test/

## ✨ 功能特点

### 📝 测试系统
- ✅ 10 道精选题目，基于 MBTI 性格理论
- ✅ 每题 4 个选项，真实反映性格倾向
- ✅ 支持上一题/下一题，可修改答案
- ✅ 必须完成所有题目才能查看结果

## 🚀 快速开始

### 第 1 步：部署到 GitHub Pages

1. 访问：https://github.com/new
2. 创建仓库：`personality-test`
3. 上传所有文件到仓库
4. 进入 Settings → Pages
5. Source 选择 `main` 分支 → Save
6. 等待 1-2 分钟，获得链接：`https://你的用户名.github.io/personality-test/`

### 第 2 步：配置 Airtable（密钥管理）

#### 2.1 注册 Airtable
- 访问：https://airtable.com/signup
- 用邮箱注册（免费计划足够用）

#### 2.2 创建 Base
1. 点击 **"+ Create"** → **"Start from scratch"**
2. Base 名称：`性格测试密钥管理`
3. 表格名称改为：`Keys`

#### 2.3 添加列
| 列名 | 类型 | 说明 |
|------|------|------|
| `Key` | Single line text | 密钥（如 EJ-ABCD1234） |
| `Type` | Single select | 类型（EJ/EP/IJ/IP） |
| `Status` | Single select | 状态（available/distributed/used） |
| `UsedAt` | Date & time | 使用时间 |

#### 2.4 获取 API 信息
1. 看 URL：`https://airtable.com/appXXXXXX/tblYYYYYY`
2. 复制 **Base ID**（app 开头）
3. 访问：https://airtable.com/settings/tokens
4. 创建 Token，勾选权限：
   - ✅ `data.records:read`
   - ✅ `data.records:write`
5. 复制 **Token**（pat 开头）

#### 2.5 修改代码
打开 `密钥管理系统.html` 和 `index.html`，找到以下配置并修改：

```javascript
const AIRTABLE_CONFIG = {
    apiKey: '你的 Token',
    baseId: '你的 Base ID',
    tableName: 'Keys'
};
```

### 第 3 步：配置收款码和客服微信

1. 准备微信收款码图片
2. 准备客服微信二维码图片
3. 上传到仓库的 `Picture/` 目录
4. 文件名：`收款码.png` 和 `客服微信.png`

### 第 4 步：修改客服微信号

打开 `index.html`，搜索 `wzsOctober`，替换为你的微信号。

## 🎯 密钥管理系统功能

### 统计卡片
- **总密钥数**：所有密钥的总数
- **可用密钥**：可以分发的密钥（绿色）
- **已分发**：已发送但未使用的密钥（橙色）
- **已使用**：已作废的密钥（红色）

### 操作按钮
- **➕ 生成密钥**：批量生成新密钥
  - 选择性格类型（EJ/EP/IJ/IP/混合）
  - 设置生成数量（1-100 个）
  - 自动生成不重复的密钥
  
- **📤 分发密钥**：随机选择一个可用密钥
  - 弹窗显示密钥
  - 一键复制
  - 自动标记为「已分发」
  
- **📥 导出密钥**：导出所有密钥数据（CSV 格式）
- **🔄 刷新数据**：从 Airtable 刷新最新数据

### 密钥列表
- 筛选：全部 / 可用 / 已分发 / 已使用
- 显示：密钥、类型、状态、使用时间
- 操作：复制、删除

## 🐛 常见问题

### Q1: 密钥管理系统显示"Invalid permissions"？
**A:** 检查 Token 权限和 Base 访问权限：
1. 访问：https://airtable.com/settings/tokens
2. 确保 Token 有 `data.records:read` 和 `data.records:write` 权限
3. 在 Token 设置中添加 Base 访问权限（点击 "+ Add a base"）
4. 选择你的 Base

### Q2: 生成密钥时报错？
**A:** 检查 Airtable 表格结构：
1. 表格名称必须是 `Keys`
2. 列名必须匹配：Key, Type, Status, UsedAt
3. 确保列类型正确（Single line text, Single select, Date & time）

### Q3: 用户输入密钥显示"密钥不存在"？
**A:** 可能的原因：
1. 密钥未生成（先去密钥管理系统生成）
2. 密钥格式错误（应该是 XX-ABCD1234）
3. Airtable 连接失败（检查 Token 和网络）

### Q4: 如何修改报告内容？
**A:** 编辑 `report.html` 文件：
1. 找到 `personalityData` 对象
2. 修改对应性格类型的内容
3. 支持 HTML 格式

### Q5: 如何修改测试题目？
**A:** 编辑 `index.html` 文件：
1. 找到 `questions` 数组
2. 修改题目和选项
3. 每个选项的 `dim` 字段决定性格维度计分
